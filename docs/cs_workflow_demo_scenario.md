# CS 품질 불량 분석 워크플로우 시연 시나리오

## 개요

본 문서는 디스플레이 제조 현장에서 발생하는 고객 품질 불량에 대한 분석 워크플로우를 시연하기 위한 가이드입니다.

### 워크플로우 단계

```
[CS 접수] → [1차 기본분석] → [귀책부서 확인] → [2차 상세분석] → [최종 보고서]
```

---

## 시나리오 1: Dead Pixel 불량 분석

### 1단계: CS 불만 접수

**상황 설명:**
APPLE 고객사에서 OLED_67_FHD 제품에서 Dead Pixel 불량이 발생하여 CS 부서에 클레임이 접수됨.

**접수 정보:**
- 고객사: APPLE
- 제품모델: OLED_67_FHD
- LOT ID: LOT20241203001
- CELL ID: CELL12345
- 결함유형: DEAD_PIXEL
- 심각도: HIGH
- 결함 설명: "화면 중앙부에 검은색 점 발견. 크기 약 0.3mm, 백색 배경에서 육안 확인 가능"

**시연 포인트:**
1. CS 접수 화면에서 불만 정보 입력
2. 접수 번호 자동 생성
3. 결함 이미지 첨부

---

### 2단계: 1차 기본 분석 (품질부서)

**분석 목표:**
- 빅데이터 기반 연관 분석
- 귀책 부서 추정
- 유사 사례 검색

**분석 내용:**

#### 2.1 빅데이터 분석
- 동일 LOT 결함 현황 조회
- 동일 제품 결함 트렌드 분석
- 설비별 불량률 분석

**예상 결과:**
```json
{
  "defect_statistics": {
    "total_count": 526,
    "severity_distribution": {
      "COSMETIC": 114,
      "MINOR": 129,
      "CRITICAL": 149,
      "MAJOR": 134
    }
  },
  "high_risk_equipment": [
    {"equipment_id": "EQ0003", "defect_rate": 9.91},
    {"equipment_id": "EQ0038", "defect_rate": 9.78}
  ]
}
```

#### 2.2 귀책 부서 판정
- 결함 유형 기반 귀책 부서 추정
- **판정 결과: TFT공정**
- 신뢰도: 78%

#### 2.3 유사 사례 검색
- CASE_DEAD_PIXEL_2024001: APPLE, ESD 원인 (유사도 89%)
- CASE_DEAD_PIXEL_2024002: SAMSUNG, particle 유입 (유사도 75%)

**시연 포인트:**
1. 1차 분석 버튼 클릭
2. 빅데이터 분석 결과 실시간 표시
3. 귀책 부서 자동 추정
4. 유사 사례 목록 표시

---

### 3단계: 귀책부서 배정

**상황 설명:**
1차 분석 결과 TFT공정이 귀책 부서로 확정되어 해당 부서에 상세 분석 요청

**배정 정보:**
- 귀책 부서: TFT공정
- 담당 분석자: Dept_Analyst_03
- 배정 일시: 2024-12-03 14:30

**시연 포인트:**
1. 귀책 부서 확정 버튼 클릭
2. 담당자 배정
3. 상태 변경 (1차분석중 → 귀책부서배정)

---

### 4단계: 2차 상세 분석 (귀책부서)

**분석 목표:**
- 결함 이미지 AI 분석
- GraphRAG 기반 원인/대책 분석
- 과거 사례 참조

#### 4.1 결함 이미지 분석 (Cosmos VLM)

**입력:**
- 결함 이미지 파일

**분석 결과:**
```json
{
  "detected_defects": [
    {
      "type": "point_defect",
      "location": {"x": 512, "y": 384},
      "size_mm": 0.3,
      "confidence": 0.92
    }
  ],
  "defect_characteristics": {
    "shape": "circular",
    "color": "dark",
    "pattern": "isolated",
    "estimated_cause": "TFT 공정 중 particle 유입 추정"
  },
  "severity_assessment": "MAJOR"
}
```

#### 4.2 GraphRAG 분석

**원인 분석:**
- 주요 원인: TFT Array 공정 중 PR 코팅 불균일
- 부가 원인:
  - Spin coater 회전속도 변동
  - Clean room 습도 관리 미흡
  - PR 소재 lot 변경

**연관 노드:**
- SPIN_COATER_01 (설비)
- PR_MATERIAL_A (자재)
- TFT_ARRAY (공정)

#### 4.3 과거 사례 분석

**참조 사례:**
| 사례 ID | 고객사 | 원인 | 대책 | 결과 |
|---------|--------|------|------|------|
| CASE_DEAD_PIXEL_2024001 | APPLE | ESD 발생 | ESD 방지 장비 점검 | 불량률 70% 감소 |
| CASE_LINE_2024003 | GOOGLE | IC 접합 불량 | COG 본딩 조건 최적화 | 불량률 80% 감소 |

**시연 포인트:**
1. 2차 분석 시작 버튼 클릭
2. 이미지 분석 결과 실시간 표시
3. GraphRAG 지식그래프 시각화
4. 과거 사례 카드 표시

---

### 5단계: 근본 원인 확정

**확정 원인:**
> TFT Array 공정 중 PR 코팅 불균일로 인한 Dead Pixel 발생.
> Spin coater 회전속도 변동 및 Clean room 습도 관리 미흡이 부가 원인으로 작용.

**원인 분석 신뢰도:** 85%

---

### 6단계: 대책 수립

#### 6.1 즉각 대책
1. Spin coater 정비 및 회전속도 재설정
2. Clean room 습도 모니터링 강화
3. PR 소재 입고검사 기준 강화
4. PM 주기 관리 강화
5. 자재 변경점 관리 프로세스 수립

#### 6.2 재발 방지 대책
1. 해당 설비 집중 모니터링 체계 구축
2. 유사 공정 라인 전수 점검 실시
3. 작업 표준서 개정 및 작업자 재교육
4. 품질 관리 기준 강화 (Spec 조정)
5. 협력업체 품질 관리 강화

---

### 7단계: 최종 보고서 생성

**출력 형식:** Word 문서 (.docx)

**보고서 구조:**
1. 불만 접수 정보
2. 결함 현상
3. 1차 기본 분석 결과
4. 2차 상세 분석 결과
5. 근본 원인 분석
6. 대책 수립
7. 결론
8. 승인

**시연 포인트:**
1. 보고서 생성 버튼 클릭
2. LLM이 과거 사례 참조하여 보고서 자동 작성
3. Word 문서 다운로드
4. Office Copilot에서 보고서 열기
5. 대화형 수정 시연

## 주요 시연 포인트

1. **빅데이터 활용**: 대량의 제조 데이터를 활용한 연관 분석
2. **AI 이미지 분석**: Cosmos VLM을 활용한 결함 이미지 자동 분석
3. **지식그래프**: GraphRAG를 활용한 원인-결과-대책 연결
4. **과거 사례 학습**: 유사 사례 자동 검색 및 대책 추천
5. **LLM 보고서 생성**: AI가 자동으로 최종 보고서 작성
6. **Office 연동**: Copilot을 통한 대화형 문서 수정

## 기술 스택

- **빅데이터 처리**: Apache Parquet, Python pandas
- **AI 이미지 분석**: NVIDIA Cosmos Reason VLM
- **지식그래프**: GraphRAG (Neo4j)
- **LLM**: Claude/GPT API
- **문서 생성**: python-docx
- **UI**: Gradio
- **Office 연동**: Microsoft Graph API
