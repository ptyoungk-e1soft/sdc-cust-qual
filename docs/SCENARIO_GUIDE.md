## 2. 시나리오 1: 고객 불만 접수 및 처리

### 2.1 시나리오 배경

**상황**: APPLE社로부터 OLED_67_FHD 모델에서 데드 픽셀(Dead Pixel) 결함이 발견되었다는 불만이 접수됨.

**목표**: 불만 접수 → 원인 분석 → 대책 수립 → 고객 회신까지 전체 프로세스 완료

### 2.2 Step 1: 불만 접수

#### 2.2.1 시스템 초기화

```
[작업] CS 워크플로우 초기화 버튼 클릭
[결과] "CS 워크플로우 시스템이 초기화되었습니다."
```

#### 2.2.2 불만 정보 입력

| 항목 | 입력값 |
|------|--------|
| 고객사 | APPLE |
| 제품모델 | OLED_67_FHD |
| LOT ID | LOT20260104001 |
| CELL ID | CELL-A1234 |
| 결함유형 | DEAD_PIXEL |
| 심각도 | HIGH |
| 결함 설명 | 화면 중앙부(좌표 512,384)에 약 0.3mm 크기의 검은색 점 발견. 정상 구동 시에도 해당 픽셀 미점등 |

#### 2.2.3 불만 접수 등록

```
[작업] 불만 접수 등록 버튼 클릭
[결과]
  - 불만접수 ID: CS-20260104-001
  - 상태: REGISTERED
  - 고객 확인 이메일 자동 생성
```

#### 2.2.4 고객 확인 이메일 발송

**자동 생성된 이메일:**
```
제목: [SDC] 품질 불만 접수 확인 - CS-20260104-001

Dear John Smith,

귀사에서 접수하신 품질 불만 사항이 정상적으로 접수되었음을 알려드립니다.

▣ 접수 정보
- 접수번호: CS-20260104-001
- 제품모델: OLED_67_FHD
- LOT ID: LOT20260104001
- 결함유형: DEAD_PIXEL (데드 픽셀)
- 심각도: HIGH

현재 당사 품질관리팀에서 해당 건에 대한 분석을 진행 중이며,
분석 완료 후 상세 결과를 회신드리겠습니다.

예상 회신 기한: 영업일 기준 5일 이내
```

**처리 절차:**
1. 이메일 내용 확인 및 수정 (필요 시)
2. 언어 변환 (영어로 번역)
3. **결재 요청** 클릭 → 결재번호 생성 (APR-20260104-001)
4. 결재 승인 대기
5. 승인 완료 후 **고객 이메일 전송** 클릭

### 2.3 Step 2: 1차 분석

#### 2.3.1 1차 분석 실행


**분석 결과:**
```markdown
## 1차 분석 결과

### 기본 정보
- 분석 ID: FA-20260104-001
- 분석 일시: 2026-01-04 14:30:00
- 분석 담당: 품질관리팀

### 결함 분류
- 결함 유형: DEAD_PIXEL (데드 픽셀)
- 결함 위치: 화면 중앙부 (512, 384)
- 결함 크기: 약 0.3mm

### 원인 추정
1. **1차 추정 원인**: TFT 소자 불량
   - TFT 게이트/소스 전극 단선 가능성
   - 박막 증착 공정 중 파티클 오염 가능성

2. **2차 추정 원인**: OLED 유기물 불량
   - 유기물 증착 불균일 가능성

### 귀책 부서 결정
- **주 귀책**: TFT공정팀 (확률 80%)
- **부 귀책**: OLED공정팀 (확률 20%)

### 긴급도 평가
- **등급**: 긴급 (Grade A)
- **사유**: 주요 고객사(APPLE) 불만, 고심각도(HIGH)

### 초기 조치 권고
1. 동일 LOT 출하 보류
2. 해당 LOT 전수 검사 실시
3. TFT 공정 파라미터 점검
```

### 2.4 Step 3: 2차 분석

#### 2.4.1 2차 분석 실행


**2차 분석 결과:**
```markdown
## 2차 상세 분석 결과

### 분석 정보
- 분석 ID: SA-20260104-001
- 분석 부서: TFT공정팀
- 분석 담당: 박영수 과장

### 근본 원인 분석 (Root Cause Analysis)

#### 5 Why 분석
1. Why: 왜 Dead Pixel이 발생했는가?
   → TFT 소자의 Gate 전극 단선

2. Why: 왜 Gate 전극이 단선되었는가?
   → 전극 박막의 접착력 부족

3. Why: 왜 접착력이 부족했는가?
   → 증착 온도가 기준보다 2°C 낮았음

4. Why: 왜 온도가 낮았는가?
   → PM 후 챔버 온도 안정화 시간 부족

5. Why: 왜 안정화 시간이 부족했는가?
   → PM 후 가동 절차에 온도 확인 단계 누락

#### 직접 원인
- CVD 챔버 PM 후 온도 미안정 상태에서 공정 진행

#### 간접 원인
- PM 후 가동 체크리스트에 온도 확인 항목 미포함
- 온도 모니터링 알람 임계값 설정 부적절

### 재발 방지 대책

| 구분 | 대책 | 담당 | 기한 |
|------|------|------|------|
| 단기 | PM 후 가동 체크리스트에 온도 확인 항목 추가 | TFT공정 | 2026-01-08 |
| 단기 | 온도 알람 임계값 조정 (±1°C) | 설비팀 | 2026-01-08 |
| 중기 | PM 후 더미 런 2회 의무화 | TFT공정 | 2026-01-15 |
| 장기 | 자동 온도 안정화 시스템 도입 검토 | 설비기술 | 2026-03-31 |

### 공정 개선 방안
1. Gate 전극 증착 온도 관리 강화 (기준 ±1°C)
2. PM 후 공정 재개 절차 표준화
3. 실시간 온도 모니터링 대시보드 구축

### 품질 관리 강화 방안
1. 출하 전 Dead Pixel 전수 검사 강화
2. 고위험 LOT 마킹 시스템 도입
3. 고객사별 품질 기준 차등 적용
```
