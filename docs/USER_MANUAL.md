## 1. 시스템 개요

### 1.1 시스템 소개

**SDC Customer Quality System**은 디스플레이 제조 품질 관리를 위한 AI 기반 통합 솔루션입니다.
본 시스템은 고객 불만 접수부터 원인 분석, 보고서 생성까지 전체 CS 워크플로우를 지원하며,
AI 기반 결함 분석과 지식 그래프를 활용한 유사 사례 검색 기능을 제공합니다.

### 1.2 주요 기능

| 기능 | 설명 |
|------|------|
| **CS 워크플로우** | 고객 불만 접수 → 1차 분석 → 2차 분석 → 최종 보고서 생성 |
| **결재 시스템** | 다단계 결재 라인, 기한 관리, 알림 발송 |
| **VLM 분석** | Cosmos Reason 7B 기반 결함 이미지 분석 |
| **GraphRAG** | Neo4j 지식그래프 기반 유사 사례 검색 |
| **빅데이터 분석** | Spark 기반 대용량 품질 데이터 분석 |
| **품질 대시보드** | 실시간 품질 현황 시각화 |

### 1.3 시스템 구성

```
┌─────────────────────────────────────────────────────────────┐
│              디스플레이 결함 분석 시스템 v2.0                 │
├─────────────────────────────────────────────────────────────┤
│   [이미지 입력] → [VLM 추론] → [GraphRAG] → [분석 결과]      │
│                    Cosmos      Neo4j                         │
│                    Reason 7B   지식그래프                    │
│                                                              │
│   [CS 불만접수] → [1차분석] → [2차분석] → [보고서생성]        │
│                    품질부서    귀책부서    최종보고           │
└─────────────────────────────────────────────────────────────┘
```

## 4. CS 워크플로우

### 4.1 개요

CS 워크플로우는 고객 불만 접수부터 최종 보고서 생성까지 전체 프로세스를 관리합니다.

```
[불만 접수] → [1차 분석] → [2차 분석] → [보고서 생성]
     ↓            ↓            ↓            ↓
  고객 이메일   품질부서      귀책부서     최종 보고서
  미팅 요청    업무 할당     상세 분석    고객 회신
```

### 4.2 Step 1. 불만 접수

#### 4.2.1 시스템 초기화

1. **CS 워크플로우 초기화** 버튼을 클릭합니다.
2. 초기화 결과가 표시됩니다.

#### 4.2.2 불만 정보 입력

| 필드 | 설명 | 예시 |
|------|------|------|
| 고객사 | 불만 접수 고객사 선택 | APPLE, SAMSUNG_MOBILE 등 |
| 제품모델 | 불만 대상 제품 선택 | OLED_67_FHD, LTPO_68_QHD 등 |
| LOT ID | 생산 LOT 번호 | LOT20241203001 |
| CELL ID | 개별 셀 번호 | CELL12345 |
| 결함유형 | 결함 유형 선택 | DEAD_PIXEL, MURA, SCRATCH 등 |
| 심각도 | 결함 심각도 | LOW, MEDIUM, HIGH, CRITICAL |
| 결함 설명 | 상세 결함 내용 | 화면 중앙부에 검은색 점 발견 |

#### 4.2.3 불만 접수 등록

1. 모든 정보 입력 후 **불만 접수 등록** 버튼 클릭
2. 불만 접수 ID가 자동 생성됩니다 (예: CS-20260104-001)
3. 고객 확인 이메일이 자동 생성됩니다

#### 4.2.4 고객 확인 이메일

| 기능 | 설명 |
|------|------|
| 이메일 내용 | 자동 생성된 이메일 확인 및 수정 가능 |
| 언어 변환 | 한국어, 영어, 일본어, 중국어 번역 지원 |
| 결재 요청 | 이메일 발송 전 결재 요청 |
| 이메일 전송 | 결재 완료 후 이메일 발송 |

#### 4.2.5 사내 미팅 요청

1. **미팅 요청 메일 생성** 버튼 클릭
2. 참석자, 예상 귀책부서, 미팅 일시가 자동 설정됩니다
3. 내용 확인 후 **미팅 요청 전송** 버튼 클릭

### 4.3 Step 2. 1차 분석

#### 4.3.1 1차 분석 실행

1. **불만 접수 ID** 입력 (Step 1에서 생성된 ID)
2. **1차 분석 실행** 버튼 클릭
3. 분석 결과 확인:
   - 결함 유형 분류
   - 원인 추정
   - 귀책 부서 결정
   - 긴급도 평가

#### 4.3.2 미팅 결과 정리

1. 미팅 노트 입력 (논의 내용, 결정 사항 등)
2. **미팅 결과 정리 (LLM)** 버튼 클릭
3. AI가 미팅 내용을 정리하여 표시

#### 4.3.3 업무 할당

| 필드 | 설명 |
|------|------|
| 담당 부서 | TFT공정, CF공정, OLED공정 등 |
| 업무 내용 | 할당할 업무 상세 내용 |
| 납기일 | 업무 완료 기한 (YYYY-MM-DD) |

1. 정보 입력 후 **업무 할당 메일 생성** 클릭
2. 내용 확인 후 **업무 할당 메일 전송** 클릭

#### 4.3.4 산출물 등록

| 산출물 유형 | 설명 |
|------------|------|
| 분석 보고서 | 1차 분석 결과 보고서 |
| 원인 조사서 | 원인 조사 결과 문서 |
| 공정 데이터 | 관련 공정 데이터 |
| 검사 결과 | 검사 결과 데이터 |
| 미팅 회의록 | 미팅 회의록 |

1. 산출물 유형 선택 및 설명 입력
2. **산출물 등록** 버튼 클릭
3. 등록된 산출물이 목록에 표시됩니다

#### 4.3.5 1차 산출물 보고서 생성

1. **1차 산출물 보고서 생성 (Word)** 버튼 클릭
2. Word 형식 보고서가 자동 생성됩니다
3. 저장 경로가 표시됩니다

#### 4.3.6 1차 분석 결재

1. **1차 분석 결재 요청** 버튼 클릭
2. 결재번호가 생성됩니다
3. 결재 의견 입력 후 **승인** 또는 **반려** 클릭

### 4.4 Step 3. 2차 분석

#### 4.4.1 2차 분석 실행

1. **불만 접수 ID** 입력
2. **1차 분석 ID** 입력 (Step 2에서 생성된 ID)
3. **2차 분석 실행** 버튼 클릭
4. 상세 분석 결과 확인:
   - 상세 원인 분석
   - 재발 방지 대책
   - 공정 개선 방안
   - 품질 관리 강화 방안

#### 4.4.2 2차 산출물 등록

| 산출물 유형 | 설명 |
|------------|------|
| 상세 원인 분석서 | 상세 원인 분석 결과 |
| 재발 방지 대책서 | 재발 방지를 위한 대책 |
| 공정 개선 계획서 | 공정 개선 계획 |
| 품질 관리 방안 | 품질 관리 강화 방안 |
| 시정 조치서 | 시정 조치 내용 |

#### 4.4.3 2차 산출물 보고서 생성

1. **2차 산출물 보고서 생성 (Word)** 버튼 클릭
2. Word 형식 보고서가 자동 생성됩니다

### 4.5 Step 4. 보고서 생성

#### 4.5.1 최종 보고서 생성

1. **불만 접수 ID** 입력
2. **결함 유형** 선택
3. **고객사** 선택
4. **최종 보고서 생성 (Word)** 버튼 클릭
5. 보고서 저장 경로가 표시됩니다

#### 4.5.2 최종 보고서 결재

1. **최종 보고서 결재 요청** 버튼 클릭
2. 결재 라인: 팀장 → 부장 → 상무
3. 각 단계별 승인 필요

#### 4.5.3 고객 회신 메일

1. **고객 회신 메일 생성** 버튼 클릭
2. 수신자 정보 자동 입력
3. 언어 변환 가능 (한국어, English, 日本語, 中文)
4. **회신 메일 결재 요청** 클릭
5. 결재 완료 후 **고객 회신 메일 전송** 클릭

### 4.6 결재 현황

#### 4.6.1 대시보드 개요

- **새로고침** 버튼으로 현황 갱신
- 요약 통계 (대기/승인/반려 건수)
- 기한 현황 (초과/긴급/경고/정상)

#### 4.6.2 결재 검색

| 검색 조건 | 설명 |
|----------|------|
| 결재번호 | 결재 고유 번호로 검색 |
| 접수ID | 불만 접수 ID로 검색 |
| 상태 | 대기/승인/반려 상태로 검색 |
| 결재유형 | 이메일/분석/보고서 유형으로 검색 |

#### 4.6.3 빠른 결재 처리

1. 결재번호 입력
2. 결재 의견 입력
3. **승인** 또는 **반려** 버튼 클릭

#### 4.6.4 기한 초과 알림 발송

- **전체 알림 발송**: 기한 초과/임박 결재 일괄 알림
- **개별 알림 발송**: 특정 결재에 대해 알림 발송
- **알림 발송 이력**: 발송 이력 조회

## 5. 빅데이터 분석

### 5.1 개요

Apache Spark 기반 대용량 품질 데이터 분석 파이프라인입니다.

### 5.2 분석 파이프라인

```
[데이터 수집] → [ETL 처리] → [분석/ML] → [결과 저장]
     ↓              ↓           ↓            ↓
   MES 데이터    정제/변환    패턴 분석    데이터마트
   검사 데이터   결합/집계    예측 모델    리포트
```

### 5.3 사용 방법

1. **파이프라인 초기화** 버튼 클릭
2. 케이스 정보 입력:
   - Case ID
   - Cell ID
   - 결함 유형
   - 발생 일자
   - 고객사
   - 심각도
   - 상세 설명
3. **분석 파이프라인 실행** 버튼 클릭
4. 분석 결과 확인

## 6. 결함 분석

### 6.1 개요

Cosmos Reason 7B VLM을 활용한 AI 기반 이미지 결함 분석입니다.

### 6.2 사용 방법

1. 분석할 이미지 업로드 (드래그 앤 드롭 또는 클릭)
2. 또는 샘플 이미지 선택
3. **분석 시작** 버튼 클릭
4. 분석 결과 확인:
   - 결함 유형
   - 심각도
   - 상세 분석 내용
   - 권장 조치사항

### 6.3 지원 결함 유형

| 결함 유형 | 설명 |
|----------|------|
| DEAD_PIXEL | 화소 불량 (검은 점) |
| BRIGHT_SPOT | 밝은 점 결함 |
| LINE_DEFECT | 라인 결함 |
| MURA | 얼룩 무라 |
| SCRATCH | 스크래치 |
| TOUCH_FAIL | 터치 불량 |

## 8. GraphRAG 관리

### 8.1 개요

Neo4j 기반 지식그래프 데이터 관리 기능입니다.

### 8.2 노드 유형

| 노드 유형 | 설명 |
|----------|------|
| Defect | 결함 정보 |
| Cause | 원인 정보 |
| Action | 조치 정보 |

### 8.3 관계 유형

| 관계 유형 | 설명 |
|----------|------|
| CAUSED_BY | 결함 → 원인 |
| REQUIRES | 원인 → 조치 |

### 8.4 사용 방법

#### 8.4.1 지식베이스 초기화

1. **지식 베이스 초기화** 버튼 클릭
2. Neo4j 연동 여부 선택
3. 초기화 결과 확인

#### 8.4.2 노드 추가

1. 노드 유형 탭 선택 (Defect/Cause/Action)
2. 필요 정보 입력
3. **추가** 버튼 클릭

#### 8.4.3 관계 추가

## 12. FAQ

### Q1. 결재 요청이 생성되지 않습니다.

**A**: 불만 접수 ID가 올바르게 입력되었는지 확인하세요. Step 1에서 생성된 ID를 정확히 입력해야 합니다.

### Q2. 이메일이 발송되지 않습니다.

**A**: 결재가 완료되었는지 확인하세요. 결재 상태가 "승인"이어야 이메일 발송이 가능합니다.

### Q3. 보고서 생성 시 오류가 발생합니다.

**A**:
1. 불만 접수 ID가 올바른지 확인
2. 이전 단계(1차/2차 분석)가 완료되었는지 확인
3. 저장 디렉토리 권한 확인

### Q4. 그래프가 표시되지 않습니다.

**A**: 지식베이스가 초기화되었는지 확인하세요. "지식 베이스 초기화" 버튼을 먼저 클릭해야 합니다.

### Q5. 분석 결과가 표시되지 않습니다.

**A**:
1. 이미지가 올바르게 업로드되었는지 확인
2. VLM 모델이 로드되었는지 확인
3. GPU 메모리가 충분한지 확인
